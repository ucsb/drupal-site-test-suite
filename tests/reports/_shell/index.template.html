Why this matters

This report shows what automated tests flagged in the custom themes, modules, and profiles shipped with this site. Automated tests catch only 30–40% of real accessibility issues — manual testing with screen readers, keyboard-only navigation, and disabled users is still required for full confidence.

Treat this report as a shared checkpoint, not a scoreboard. Issues grouped by user impact tell you who is affected and why; severity tells you how soon to address it. Each issue includes the rule ID, file path or selector, and tool-native message for direct triage.
